# ☕ Bean Scene — Git & Code Review Workshop

Лендінг-сторінка вигаданої кав'ярні. Простий HTML + CSS проєкт, на якому ми будемо вчитися працювати з Git, гілками, Pull Request'ами і code review.

## 🎯 Мета воркшопу

Кожен учасник повинен:

1. Створити свою гілку
2. Вибрати одне завдання зі списку нижче
3. Внести зміни і зробити коміт
4. Запушити гілку і відкрити Pull Request
5. Прорев'юити PR одного з колег

## 🚀 Як почати

### 1. Клонуйте репозиторій

```bash
git clone <посилання-на-репо>
cd workshop-project
```

### 2. Відкрийте `index.html` у браузері

Просто двічі клікніть на файл — він відкриється в браузері. Жодного `npm install` чи серверу не потрібно.

### 3. Створіть свою гілку

```bash
git checkout -b feature/ваше-імʼя
```

Наприклад: `feature/anna-fix-footer` або `feature/petro-add-raf`.

## 💾 Як зробити коміт і пуш

```bash
# Перевірити що змінилось
git status
git diff

# Додати зміни і закомітити
git add .
git commit -m "Fix typo in Instagram link"

# Запушити свою гілку
git push origin feature/ваше-імʼя
```

**Хороший commit message:**
- ✅ `"Fix typo in Instagram link"` — імператив, конкретно
- ✅ `"Add Raf coffee to menu"` — зрозуміло що додано
- ❌ `"fix"`, `"updates"`, `"WIP"` — нічого не пояснює

## 🔍 Як відкрити Pull Request

1. Зайдіть на сторінку репозиторію на GitHub
2. Має з'явитись жовтий банер `Compare & pull request` — натисніть
3. Заповніть **заголовок** і **опис**:
   - Що ви змінили
   - Чому саме так
   - Як це перевірити (наприклад: "відкрийте index.html і подивіться на footer")
4. Натисніть **Create pull request**

## 👀 Як рев'юити чужий PR

1. Зайдіть на вкладку **Pull requests** репозиторію
2. Відкрийте PR колеги
3. Подивіться вкладку **Files changed**
4. Залишіть **мінімум 2 коментарі**, використовуючи теги:
   - `blocker:` — критично, не можна мерджити
   - `suggestion:` — можна зробити краще
   - `nit:` — дрібниця, на ваш смак
   - `question:` — просто хочу зрозуміти
   - `praise:` — вдале рішення

**Приклади коментарів:**

- 👍 `praise: чисто і коротко, дякую!`
- 💡 `suggestion: можна винести колір у CSS-змінну, щоб не повторювати`
- ❓ `question: чому 14px, а не 16px як в інших місцях?`

## 🧪 Як перевірити свої зміни

Просто відкрийте `index.html` у браузері (або оновіть сторінку через F5). Жодного збирання не потрібно.
